HTML

HTML: Structure, meaning, and content

HTML stands for HyperText Markup Language. It is the language used to structure web content. Instead of focusing on decoration, HTML focuses on meaning. For example, a heading should use a heading element, a paragraph should use a paragraph element, and navigation links should appear inside a navigation area.

A valid HTML5 document starts with <!DOCTYPE html>. The main root element is <html>. Inside that, the <head> contains metadata such as the page title, linked CSS files, and viewport settings, while the <body> contains the visible content shown to the user.

HTML elements can also contain attributes. Attributes provide extra information about an element. For example, an anchor uses href to store its destination, an image uses src to define the file location, and an image should also use alt text so users understand the image if it cannot be seen.

Good HTML matters because semantic structure improves accessibility, readability, and maintenance. Screen readers rely on meaningful headings and landmark elements, search engines benefit from clear document structure, and developers can understand and update the page more easily when the markup is organised around meaning instead of visual tricks.

Important HTML idea: HTML should describe the role and meaning of content, not just the way it should look.

Document Structure

Use <!DOCTYPE html>, <html>, <head>, and <body>.

Semantic Tags

Use <header>, <nav>, <section>, <article>, and <footer>.

Attributes

Common attributes include href, src, alt, id, and class.

Links & Images

Use <a> for hyperlinks and <img> for images with meaningful alt text.

Lists & Tables

Use lists for grouped items and tables for tabular data rather than page layout.

Forms

Use <form>, <label>, <input>, and validation attributes like required.

HTML Example: Basic Page Structure

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>My Web Page</title>
</head>
<body>
  <header>
    <h1>Web Foundations</h1>
  </header>

  <main>
    <section>
      <p>HTML gives content structure and meaning.</p>
    </section>
  </main>
</body>
</html>

HTML Example: Links, Images, and Lists

<nav>
  <a href="quiz.html">Go to Quiz</a>
</nav>

<img src="images/web-dev.gif" alt="Animated web development illustration">

<ul>
  <li>HTML structures content</li>
  <li>CSS styles the page</li>
  <li>JavaScript adds behaviour</li>
</ul>

HTML Example: Form and Validation

<form>
  <label for="email">Email</label>
  <input type="email" id="email" required>

  <label for="age">Age</label>
  <input type="number" id="age" min="18" max="100">

  <button type="submit">Submit</button>
</form>
Feature Good HTML Practice Poor HTML Practice
Page structure Use semantic sections like <header> and <main> Use only generic <div> elements everywhere
Images Add meaningful alt text Leave out alt text
Forms Match labels to inputs and use validation attributes Use inputs without labels or validation
Links Use clear link text like “Go to Quiz” Use vague text like “Click here” everywhere
Why are labels and validation important?

Labels make forms clearer and easier to use, while native HTML validation can help check values such as email format, ranges, and required fields before the form is submitted.

CSS

CSS: Style and presentation

CSS stands for Cascading Style Sheets. It controls presentation, including colours, spacing, borders, positioning, and responsive layout. A key principle is separating content from presentation: HTML provides meaning, while CSS controls appearance.

CSS rules are made from selectors and declarations. A selector chooses which elements to style, and the declarations define properties and values. External stylesheets are very useful because one stylesheet can style multiple pages consistently.

Following CSS best practices improves maintainability. Reusable classes, consistent spacing, and responsive layout rules help pages look professional across different screen sizes and make later updates much easier than repeating inline styles throughout the HTML.

Important CSS idea: one stylesheet can keep multiple pages visually consistent.

CSS Example: Styling a Page

body {
  font-family: Arial, sans-serif;
  background-color: #f7f9fc;
  color: #1f2937;
}

h1 {
  color: #1f4e79;
}

.card {
  padding: 1rem;
  border-radius: 10px;
}

CSS Example: Linking an External Stylesheet

<head>
  <link rel="stylesheet" href="css/style.css">
</head>

JavaScript

JavaScript: Interactivity and behaviour

JavaScript runs in the browser and reacts to user actions such as clicks, typing, scrolling, and form input. It can select parts of the page and update the document without reloading the whole page.

This is often done through the DOM, which represents the page as a tree of objects. JavaScript can use that model to change text, styles, classes, or even create new elements while the page is running.

Good JavaScript is not only about making something happen. It should also be readable, testable, and safe to extend. Clear function names, separated logic, and careful event handling make debugging easier and reduce the chance of small mistakes breaking the whole page.

Important JavaScript idea: JavaScript is event-driven and can update the page instantly.

JavaScript Example: Click Event

document.getElementById("demoBtn").addEventListener("click", function () {
  document.getElementById("demoText").textContent =
    "JavaScript updated this content.";
});
